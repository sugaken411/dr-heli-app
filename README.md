# 🚁 AW109 EMS System (v11.0)

航空医療・ドクターヘリ運用のための総合DXプラットフォーム。
現場でのオフライン入力、電子カルテへの転記、デブリーフィング、そして多職種での事案振り返りをシームレスに連携します。

## 🌟 v11.0 の主なアップデート内容
* **UI/UXの完全統一:** iOSライクなフラットデザイン、全角・半角の自動補正、テンキーの自動立ち上げを全アプリに適用。
* **入力のスマート化:** 現場診断（病名ジェネレーター）やキーワード要請を「チップ式」のワンタップ入力に改修。
* **デブリーフィング連携:** 事案DBとデブリーフィングDBの双方向連携を強化し、過去の事例を「時間・スキーム・タグ」で直感的に検索・振り返り可能に。
* **統計ダッシュボード (BI):** 個人実績のフィルタリング、時間帯ヒートマップ、疾患・処置ランキングの自動集計機能を搭載。

## 📂 アプリケーション構成
1. `portal.html` - 全アプリの玄関口（未完了アラート・お知らせ表示）
2. `index.html` - 症例登録アプリ（現場入力用）
3. `search.html` - 事案検索コンソール（カルテ完了処理・データ修正）
4. `report.html` - 業務日報（浜町日誌・タイムライン自動生成）
5. `debriefing.html` - デブリーフィング（多職種振り返り・評価入力）
6. `debrief_search.html` - 教育・事案検索（過去事例の学習用アーカイブ）
7. `library.html` - 資料集・アーカイブ（ガイドライン等の検索）
8. `summary.html` - 統計ダッシュボード（KPI集計・CSV出力）
9. `admin.html` - システム管理コンソール（マスタデータの追加・Q&A回答）

## ⚙️ 技術仕様・連携
* **バックエンド:** Google Apps Script (GAS)
* **データベース:** Google Spreadsheet
* **フロントエンド:** HTML5 / CSS3 (Tailwind CSS) / Vanilla JS (一部 React/Babel 使用)
* **PWA対応:** `manifest.json` と `sw.js` により、iPad/スマートフォンでのスタンドアロン起動とオフラインキャッシュに対応。
